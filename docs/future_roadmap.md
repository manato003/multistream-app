# Multistream Nexus - 次回作業用メモ (Future Roadmap)

ユーザーからのフィードバックに基づいた、今後の改善・拡張予定リストです。

## 1. ビジュアル & UI 改善
- [ ] **プラットフォームアイコンの刷新**
  - Twitch/YouTube の公式ロゴまたは類するデザインを使用し、一目で判別可能な切り替えボタンにする。
- [ ] **ガイドモーダルの追加**
  - ヘッダーに「？」アイコンを配置。利用方法（ドラッグ、ダブルクリック拡大等）をガイドするモーダルを表示。

## 2. レイアウト & 操作性の高度化
- [ ] **ドラッグ入替の最適化**
  - 現在の実装のレスポンスを改善し、より滑らかで確実な入れ替えを実現する。
- [ ] **動的リサイズシステム (Tiling Window Manager風)**
  - 任意の枠のサイズを変更した際、周囲の枠も自動で追随してリサイズされる仕組みの導入。
- [ ] **レイアウト・プリセット**
  - 上下分割、縦分割、横分割など、特定の構成をワンクリックで適用できる機能。

## 3. 機能拡張
- [ ] **コメント欄（チャット）表示機能**
  - 各配信の横にチャット欄を表示できるオプション。
- [ ] **設定情報のインポート/エクスポート (コード式)**
  - バックエンドなしでセッションを共有・復旧する仕組み.
  - 現在の配置情報をテキストデータ（設定コード）として生成・表示。
  - 次回利用時にそのコードを入力することで、前回と同じ配信枠・配置を即座に復元。
  - 配信追加モーダルに入力欄を、ヘッダーにコード生成ボタンを設置。

## 4. レイアウト・アルゴリズムの調整
- [ ] 4, 6, 9 画面など、特定の数において常に最適な「フィル」形式になるよう、さらに計算式をブラッシュアップする。
